# Libretro 81 core

!!! warning "Work in Progress :material-progress-wrench:"
    This core is currently a **work-in-progress** and undergoing very active
    development. Information on this page is subject to change without notice,
    and may not reflect the most recent updates to the core.

The Libretro 81 core is a work-in-progress port of the EightyOne emulator (a/k/a
THE Sinclair Emulator) to the Libretro API. The classic ZX81 games are all over
the Internet, but check the colorized folder for games with Chroma 81 support.
There are also many original games for the ZX81; check out [Bob's
Stuff][bobs-stuff] for some high quality games.

## Author/License

## Emulated hardware

!!! info inline end "Gamer lingo"
    Over the years the Sinclair ZX81 has acquired the nickname "Zeddy" which
    can prove to be a useful term when searching for information or content
    related to it.

EightyOne emulates the Sinclair ZX80 and ZX81 consoles and many of their
prolific clones, as well as other computers based on the same hardware:

- Sinclair ZX80
- Sinclair ZX81
- Timex TS1000
- Timex TS1500
- Jupiter ACE
- Lambda 8300
- MicroDigital TK85
- Ringo R470

However, 81-libretro only emulates the Sinclair ZX81 with 16KiB RAM for now.
Other machines will be added as time permits, and pull requests enabling more
emulation states are most welcome.

EightyOne also emulates some ZX Spectrum machines, but those were left out of
this core on purpose.

## File extensions

Content that can be loaded by the 81 core has one of the following file
extensions:

- `.p`
- `.tzx`
- `.t81`

The core correctly loads and runs some games in the "`.p`" file format, while
the support for the "`.tzx`" is also available.

### Filesystem directories

The EightyOne core's directory name is 'EightyOne'.

### Geometry and timing

The Libretro 81 core provides&hellip;

- &hellip;a video refresh rate of 50 frames/second.
- &hellip;an audio sample rate of 44100 Hz.

### Auto-configuration

Since configuring the core for each game can be a tedious task, the EightyOne
core features auto-configuration. Games that support auto-configuration are
listed in the [src/gamedb/gamedb.json file][gamedb.json], along with some
information and the configuration required to play them.

Currently, there's no way to change the auto-configuration settings short of
recompiling the core after making the changes. If you feel the provided
auto-configuration could be better or has bugs, please create a ticket on the
[Issue Tracker][81-issues].

### Colorization

Colorization works by loading a program prior to loading the game. This program
will check if the Chroma 81 expansion is installed, and, if it is, configures
the colors of the ZX81 characters, and then loads the original game, which runs
unaware that it's playing with colors.

Since the EightyOne core can't load arbitrary programs from the file system, the
colorization program and the game must exist in the same file.

## Input devices

The Libretro 81 core supports the following device type(s) in the controls menu;
device types shown in **boldface** are the default for the specified user(s).

### User 1

|            Device           |                     Type                       |
|:---------------------------:|:----------------------------------------------:|
|            None             |               Input disabled                   |
|        **RetroPad**         |                   Joypad                       |
| Cursor Joystick { .middle } | Joypad<br />(switch to this for joypad inputs) |

### User 2

!!! tip inline end "Keyboard inputs"
    Switch the User 1 device type to None if the correct keyboard inputs aren't
    being read when set to `Sinclair Keyboard`.

|             Device            |                        Type                        |
|:-----------------------------:|:--------------------------------------------------:|
|              None             | Input disabled                                     |
|          **RetroPad**         | Joypad                                             |
| Sinclair Keyboard { .middle } | Keyboard<br />(switch to this for keyboard inputs) |

### Controller tables

#### Joypad

!!! attention inline end "Joystick inputs"
    Use the Joypad mapping core options to configure the Cursor Joystick inputs.
    The defaults are the keyboard cursor keys for the directional pad and ++0++
    to all the other buttons.
